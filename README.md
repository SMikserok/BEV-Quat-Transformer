Программный комплекс для генерации ориентированного Bird-Eye-View (BEV) изображения на основе кватернионной математики. Проект разработан для автоматизации трансформации перспективных изображений с камер дронов/автомобилей в ортогональную проекцию (вид строго сверху).

Ориентированные BEV-кадры критически важны для разметки дорожного полотна, обучения систем помощи водителю (ADAS) и беспилотного транспорта.

## Ключевые особенности
- Продвинутая математика: Использование кватернионов для устранения пространственных искажений (крен, тангаж, рыскание) и предотвращения проблемы шарнирного замка (Gimbal Lock).
- Высокоскоростная батч-обработка: Аппаратное ускорение трансформации изображений на видеокартах (GPU) с использованием torch.nn.functional.grid_sample.
- Умная защита от OOM: Встроенный механизм динамического масштабирования batch_size. Если видеокарте не хватает памяти (CUDA Out of Memory), скрипт автоматически очистит кэш, уменьшит батч и продолжит работу без падения.
- Изоляция конфигурации: Все настройки путей, размеров выходных изображений и углов обзора (FOV) вынесены в удобный config.yaml (OmegaConf).

## Установка и развертывание

Проект использует современный и сверхбыстрый пакетный менеджер uv для управления зависимостями.

1. Склонируйте репозиторий:
```bash
git clone https://github.com/SMikserok/BEV-Transformation.git
cd BEV-Transformation
```

2. Установите пакетный менеджер uv (если еще не установлен):
* Linux/macOS: `curl -LsSf https://astral.sh/uv/install.sh | sh`
* Windows: `powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"`

3. Установите зависимости и создайте окружение:
```bash
uv sync
```
Эта команда автоматически создаст виртуальное окружение (.venv) и скачает все библиотеки, указанные в файле pyproject.toml.

## Настройка конфигурации

В корне проекта создайте файл config.yaml (если его нет) и укажите пути к вашим датасетам. Пример базовой конфигурации:

```yaml
pan_to_bev:
  images_to_process_limit: 1
  json_path: "/path/to/information.json"
  images_dir: "/path/to/images"
  panorama_dir: "/path/to/result/panorama"
  bev_dir: "/path/to/result/bev"
  panorama_width: 1024
  panorama_height: 512
  bev_size: 512
  fov: 120.0
  batch_size: 8
  device: "cpu" # Может быть "cpu", "cuda", или "auto"

image_eller_quat:
  image_path: "/path/to/images/4_1732853237746729.jpg"
  output_width: 1024
  output_height: 512
  quaternion: [0.13654, 0.0103153, 0.13654, -0.981109]
  horizontal_fov_deg: 90.0
  output_path: "/path/to/result/panorama_inverse.png"
```

## Использование

### 1. Массовая обработка датасета (Пайплайн)
Запускает основной конвейер. Читает телеметрию из JSON, выравнивает горизонт для каждого кадра и генерирует BEV-проекции с ускорением:
```bash
uv run pan_to_bev_v2.py
```

### 2. Одиночная отладка математики
Используется для тестирования применения конкретного кватерниона к одной картинке (без батчей и пайплайна):
```bash
uv run image_eller_quat.py
```

## Структура проекта
* pan_to_bev_v2.py — главный модуль пайплайна, отвечает за логистику данных, генерацию UV-сетки вида сверху и тензорный ремаппинг (PyTorch).
* image_eller_quat.py — математическое ядро, отвечает за перевод кватернионов в матрицы вращения и проекцию сфер в декартову систему (OpenCV).
* test_project.py — Unit-тесты для проверки математики и устойчивости к ошибкам.
* config.yaml — конфигурационный файл (настройки путей и параметров).
* pyproject.toml — конфигурация зависимостей пакетного менеджера uv.
